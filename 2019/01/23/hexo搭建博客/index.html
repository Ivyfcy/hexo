hexo搭建博客

2019年01月23日 |
hexo
| hexo github yml

介绍hexo

  • 是一个快速而强大的静态博客生成框架，它基于Node.js

安装前准备工具

  • node.js
  • git

安装命令

  • npm install -g hexo-cli
    • hexo-cli会安装hexo同时安装一些命令，就可以通过命令来操控hexo
  • npm init ###
    • 在指定文件夹中新建所需的文件（各种需要的包和文件）—为什么需要git，因为有一个github hexo-starter 把hexo所有的包文件都放在其中）
  • cd
  • npm install
  • hexo generate 生成静态文件网页（hexo g）
  • npm server 启动服务器
    • 简写hexo s 访问网址为：https：//localhost：4000/
    • 如果想更改端口，可执行时使用-p 选项指定其他端口 hexo server -p 5000

更换主题 — Aath

  • 跳转到相对应的github项目下

安装下载主题

  • git clone -b master https://github.com/lewis-geek/hexo-theme-Aath.git themes/aath 把主题下载到themes/aath文件夹下

依赖

  • npm install –save hexo-renderer-sass

配置

  • hexo 有两个配置文件，一个在hexo项目的根目录，另一个在主题文件夹的根目录，文件名都为_config.yml

项目配置

主题配置

About 页面

  • 需要生成 about 页面，需要在 source 目录下创建一个 about 目录，并在 about 目录下创建一个 index.md 文件，为该文件添加 layout: about 属性

Tags 页面

  • 如果需要生成 tags 页面，需要在 source 目录下创建一个 tags 目录，并在 tags 目录下创建一个 index.md 文件，为该文件添加 layout: tags 属性

文章列表

文章图片

  • 在markdown文件顶部添加 img 属性，属性值为图片地址

    文章描述

  • 在markdown文件顶部添加 description 属性，可以在首页文章列表显示文章的描述，如果没有这个属性，会自动截取文章内容作为文章的描述

    注意点：img属性和description属性都没有链接跳转功能，注意中英文符号

语言设置

  • 因为在Aath主题下有些是英文有些是中文，很突兀
  • 在根目录_config.yml文件里找到 language 属性，填上当前主题里你想要的语言 如：zh-CN

设置实时更新

  • 发现一个移植插件：hexo-browsersync
  • npm install hexo-browsersync –save
  • 然后运行 hexo server 即可

    注意：这个插件只是实时更新文章，并不会实时更新_config.yml配置

yml介绍

  • 它是一种直观的，容易被人类阅读，以及被机器识别的数据格式。
  • 类似xml，但是语法比XML简单很多

    yml基本语法

    基本语法

  • k:(空格)v 表示一对键值对(空格必须有)

  • 以空格的缩进来控制层级关系，只要是左对齐的一列数据，都是同一个层级的

  • 属性和值都是大小写敏感

值的写法

  • 字面量：普通的值（数字，字符串，布尔）
  • 字符串默认不用加上单引号或者双引号
  • “” 双引号，不会转义字符串里面的特殊字符；特殊字符会作为本身想表示的意思
  • ‘’ 单引号，会转义特殊字符，特殊字符最终只是一个普通的字符串数据

对象，Map（属性和值）（键值对）

  • k: v：在下一行来写对象的属性和值的关系；注意缩进

  • 对象还是k：v 的方式

  • 行内写法

数组 （List，Set）

  • 用 - 值表示数组中的一个元素

新建

  • hexo new //新建文章
  • hexo new page “about” //新建页面;同时可以在主题根目录下的_config.yml配置新的路由入口

部署

  • 配置 deploy 属性在根目录下

  • 配置 root 属性 就是 github仓库的名

  • npm install hexo-deployer-git –save //安装 hexo-deployer-git

  • hexo clean // 清除缓存文件 (db.json) 和已生成的静态文件 (public)；在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。
  • hexo g //生存静态文件
  • hexo d //部署网站
